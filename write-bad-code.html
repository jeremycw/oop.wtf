Write Bad Code

2019-03-06

or: How I Learned to Code the Thing

  • what is good code?
    • code follows some set of dogmatic ‘principals’
    • pure, no side effects
    • pleases our need to quantify maintainability
    • architecture
  • people ask how to write good code
    • how to organize scrabble: ecs
  • people tell you how to write good code
    • jon blow comment on hacker news, reply saying enitity inheritance
  • good code is secretly bad code and bad is good
    • correct abstractions arent obvious, ‘good’ code is worse
    • no plan ever survives first contact
    • a lot of code that doesn’t do the thing
    • more code is worse
  • look at the code of 10x developers, it’s ‘bad’
    • writing ‘good’ code wastes a bunch of time
  • how to write bad code
    • forget conventional wisdom and write code
    • just write the code to do the thing
    • reevaluate code every time it doubles
    • make changes when you feel the pain
    • abstractions and interfaces from necessity not cargo cult
